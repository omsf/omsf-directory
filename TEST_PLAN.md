# Test Plan

The purpose of this document is to capture what our test plan is when reviewing for breaking changes to the UI.

## Areas of interest

- [ ] Do all cards appear on page?
- [ ] Do drop downs appear for tag filtering?
- [ ] Can you actually apply filters?
- [ ] Are we able to switch between all menu items?
- [ ] Is the form able to be validated correctly when filling out all fields?

## Process

1. Go through each area of interest (see above).
2. Validate that everything works as expected
3. Compare against the base site currently deployed at [https://directory.omsf.io](https://directory.omsf.io).
